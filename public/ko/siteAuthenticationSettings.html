연결된 클라이언트
사용자 로그인 자격 증명을 처음 제공한 후 모바일 앱, 동기화 클라이언트 등과 같은 Tableau 클라이언트가 이 사이트에 인증된 상태로 유지될 수 있습니다. 이 경우 사용자의 자격 증명을 저장하지 않고 각 사용자와 클라이언트를 고유하게 식별하는 보안 토큰을 사용하여 연결이 설정됩니다.
이 Tableau Online 사이트에서 연결된 클라이언트를 허용합니다.
이 사이트의 인증
TableauID
모든 사이트의 기본 인증 유형
SAML 기반의 Single Sign-On - SAML을 사용하도록 설정 - SAML을 사용하지 않도록 설정
Single Sign-On을 사용하도록 설정하려면 SAML을 지원하는 ID 공급자(IdP)와, IdP의 이메일 주소에 매핑되는 각 사용자의 이메일 주소가 있어야 합니다.
 
Single Sign-On을 위한 SAML을 사용하려면 아래 단계를 수행하십시오.
1Tableau Online에서 메타데이터 파일 내보내기
IdP에 필요한 메타데이터를 가져오기 위한 옵션을 선택합니다.
● 모든 필요한 메타데이터를 포함하는 단일 XML 파일을 저장합니다. 
또는
● Tableau Online 엔터티 ID와 ACS URL을 개별적으로 복사하고 X.509 인증서를 CER 파일로 다운로드합니다.
Tableau Online 엔터티 ID
ACS(Assertion Consumer Service) URL
2외부 단계: IdP로 메타데이터 파일 가져오기
ID 공급자(IdP)의 사이트로 이동한 후 제공되는 지침에 따라 SAML을 구성하십시오. IdP에 따라 1단계에서 내보낸 메타데이터 파일을 가져오거나 엔터티 ID, ACS URL 및 인증서를 개별적으로 추가해야 합니다.
3외부 단계: IdP에서 메타데이터 파일 내보내기
IdP에서 메타데이터 파일을 내보내거나 고유한 메타데이터 파일을 생성합니다.
4Tableau Online로 메타데이터 파일 가져오기
IdP 엔터티 ID
SSO 서비스 URL
IdP가 SAML 단일 로그아웃(SLO)을 지원하도록 구성되어 있습니다.
IdP가 SAML 단일 로그아웃(SLO)을 지원하도록 구성되어 있지 않습니다.
5어설션 일치
어설션에는 사용자에 대한 인증 및 기타 특성이 포함됩니다. 여기에서 Tableau Online에 필요한 정보가 포함된 IdP 어설션 이름을 지정할 수 있습니다. 어설션 이름은 IdP의 SAML 구성에서 확인할 수 있습니다.
Tableau Online 특성ID 공급자(IdP) 어설션 이름
이메일
인증에 성공하려면 이메일 주소가 일치해야 합니다.
표시 이름
IdP에 정보가 저장되는 방식에 따라 이름과 성에 대한 어설션 이름을 제공하거나, 전체 이름에 대한 어셜선 이름을 제공하십시오. Tableau Online에서는 이러한 어설션을 사용하여 표시 이름을 설정합니다.

6사용자 선택
SAML로 로그인할 수 있는 사용자를 지정합니다.
사용자 보기 - Tableau Online 사용자 목록에서 사용자를 선택합니다.
사용자 추가 - 새 사용자의 이메일 주소를 입력합니다.
7Tableau Online를 사용하여 인증
이제 승인된 사용자가 Single Sign-On을 사용하여 Tableau Online에 액세스할 수 있습니다.
사용자에게 인증 문제가 발생한 경우 다음 작업을 수행하여 문제를 해결할 수 있습니다.
● 어설션 일치 문제 해결(5단계) 
● 온라인에서 IdP로 데이터, 인증서 다시 내보내기(1-3단계)
● 새 메타데이터 파일 다시 로드(4단계)
